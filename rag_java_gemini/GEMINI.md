# Java Code Q&A RAG Agent プロジェクト・ステアリング

## 開発ルール
- **SPEC（仕様書）駆動開発**: `spec/` ディレクトリ配下に詳細仕様を作成し、それに従って実装する。
- **TDD（テスト駆動開発）**: テストを先に作成し、テストが通ることを確認しながら実装を進める。
- **パッケージ管理**: `uv` を使用する。`pip` の直接使用は禁止。
- **型ヒント**: 全ての関数・クラスに Python の型ヒント（`typing` / `pydantic`）を付与する。
- **コード品質**: `ruff` による Lint/Format、`mypy` による型チェックを維持する。
- **言語**: 日本語で応答し、ドキュメントも日本語で作成する。
- **自己デバッグ**: エラーが発生した場合は自律的にデバッグし、解決策を提示する。
- **要件確認**: 不明点がある場合は実装前にユーザーに質問する。

## SPEC修正ルール（バイブコーディング防止）
仕様の変更や追加が必要な場合、以下の手順を厳守する：
1. `spec/` 内の該当する SPEC ファイルを修正する。
2. 修正内容をユーザーに提示し、「SPECシートを修正しました。確認してください。OKであれば実装します。」と確認を得る。
3. ユーザーの承認後、SPEC に従って実装・テストを行う。

## 環境変数・秘密情報管理
- 秘密情報は `.env` ファイルで管理し、`python-dotenv` で読み込む。
- `.env` は絶対コミットせず、`.gitignore` に含める。
- `.env.example` を作成し、必要な設定項目を明示する。

## 技術スタック
- **言語**: Python 3.10+
- **RAG**: LangChain, ChromaDB, Ollama
- **Models**: `qwen2.5-coder:7b` (LLM), `nomic-embed-text` (Embedding)
- **CLI**: Typer
- **Task Runner**: Makefile
