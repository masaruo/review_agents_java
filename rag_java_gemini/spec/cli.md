# SPEC: CLI (cli.md)

## 1. 概要
`typer` を使用した CLI インタフェース。すべてのコマンドは `java-qa` エントリポイントを通じて実行される。

## 2. サブコマンド

### 2.1. `index`
- **引数**:
  - `--project` / `-p`: プロジェクト名 (必須)
  - `--path`: インデックス対象ディレクトリ (必須)
- **挙動**:
  1. `path` が有効か確認。
  2. プロジェクトを登録。
  3. インデックス構築を開始。
  4. 完了時に「Index built successfully for project: {name}」を表示。
- **異常系**:
  - `path` 不在: エラー表示。
  - Ollama 接続不可: エラー表示。

### 2.2. `chat`
- **引数**:
  - `--project` / `-p`: プロジェクト名 (必須)
- **挙動**:
  1. プロジェクトの登録有無を確認。
  2. インデックスの有無を確認。
  3. 対話ループを開始。
  4. ユーザーからの入力を受け付け、回答を表示。
  5. `exit` / `quit` で終了。
- **異常系**:
  - プロジェクト未登録: 登録済みリストを表示してエラー。
  - インデックス未構築: `make index` を促してエラー。

### 2.3. `list`
- **挙動**:
  - 登録済みのプロジェクト名、パス、インデックス状況を表形式で表示。

### 2.4. `delete`
- **引数**:
  - `--project` / `-p`: プロジェクト名 (必須)
- **挙動**:
  1. プロジェクト登録を解除。
  2. インデックスディレクトリを削除。
  3. 完了メッセージを表示。

## 3. 共通オプション
- `--config`: 設定ファイルへのパス (デフォルト: `./config.yaml`)

## 4. エラー表示 (STDERR)
- **Ollama 未起動**: 接続先 URL、エラー種別、推奨対処法を出力。
- **モデル未取得**: 必要な `ollama pull` コマンドを出力。
- **権限エラー**: ディレクトリ作成等の失敗時に理由を出力。
