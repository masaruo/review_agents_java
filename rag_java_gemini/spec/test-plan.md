# SPEC: テスト計画 (test-plan.md)

## 1. 概要
テスト駆動開発（TDD）を実践するためのテストケースと実施方法を定義する。

## 2. カテゴリ別テストケース

### 2.1. 単体テスト (Unit Tests)
- **Indexer**:
  - `FileScanner`: 正しいパスから `.java` ファイルを抽出できるか。
  - `Chunker`: メソッド単位の分割が正しく行われるか、メタデータが付与されているか。
- **Retriever**: 類似度に基づき正しいチャンクが取得できるか。
- **ProjectManager**: プロジェクトの追加、削除、取得が正しく動作するか。
- **Config**: `config.yaml` の読み込みと Pydantic バリデーションが機能するか。

### 2.2. 統合テスト (Integration Tests)
- **RAG Chain**:
  - 検索から LLM 回答までの一連の流れ（モックを含む）。
  - 会話履歴が次回のプロンプトに反映されるか。
- **CLI**:
  - `index` サブコマンド実行後のデータ永続化。
  - `list` コマンドで登録済みプロジェクトが表示されるか。

### 2.3. エラーハンドリングテスト
- Ollama 未起動時の例外キャッチとメッセージ出力。
- インデックス未構築時の `chat` コマンド動作。
- 存在しないプロジェクト指定時のエラー。

### 2.4. リソース・制限テスト
- `max_input_tokens` 超過時の切り詰めロジック。
- 非常に大きい `.java` ファイルのチャンキング。

## 3. テスト環境・ツール
- **pytest**: 基本テストフレームワーク。
- **pytest-mock**: 外部依存（Ollama API 等）のモックに使用。
- **fixtures/sample_java/**: テスト用の Java ソースコードディレクトリ。

## 4. 実行方法
```bash
make test
```
すべてのテストがパスするまで、実装を完了とみなさない。
