# SPEC: コンポーネント (components.md)

## 1. FileScanner
- **機能**: 指定されたディレクトリから `.java` ファイルを再帰的に検索。
- **入力**: `/path/to/project`
- **出力**: List of `file_path`
- **フィルタ**: `src/` 配下のみを対象とし、`.java` 拡張子以外は無視。

## 2. Chunker
- **機能**: `.java` ファイルを論理的な単位（メソッド単位）で分割。
- **アルゴリズム**:
  - ファイル全体が 1,000 トークン未満の場合は、ファイル単位。
  - 1,000 トークン以上の場合は、メソッド単位に分割。
- **付与メタデータ**:
  - `file_path`: 元ファイルの相対パス。
  - `imports`: 関連するインポート文。
  - `class_signature`: クラスの宣言行。
  - `member_variables`: クラス内のメンバ変数。
  - `chunk_type`: `method` または `full_file`。
- **実装上の注意**: 正規表現または簡易的なパーサを使用して、メソッドの開始・終了を判定。

## 3. Embedder (Ollama Embeddings)
- **機能**: テキストをベクトル化。
- **モデル**: `nomic-embed-text`
- **バッチ処理**: インデックス構築時は、複数のチャンクをまとめて Ollama へリクエスト（リクエスト回数の削減）。
- **エラー処理**: Ollama 接続不可時は即時エラー。

## 4. Vector DB (ChromaDB)
- **機能**: ベクトルとメタデータの永続化保存。
- **保存先**: `~/.java_qa_agent/indexes/{project_name}/`
- **操作**: 
  - `add`: チャンクの追加。
  - `query`: 類似度による検索。
  - `delete`: プロジェクト単位の全データ削除。

## 5. Retriever
- **機能**: ユーザーの質問に対して類似するチャンクを取得。
- **入力**: ユーザー質問（ベクトル化済み）。
- **出力**: List of (チャンクテキスト, メタデータ, 類似度スコア)。
- **制約**: `top_k` 件を取得。

## 6. Context Builder
- **機能**: LLMに渡す最終的なプロンプトを作成。
- **入力**:
  - 検索結果チャンク。
  - 会話履歴（直近 `max_history_turns` 分）。
  - ユーザーの最新の質問。
- **トークン制限管理**: `max_input_tokens` を超えないように、履歴および検索結果を古い順または重要度の低い順から切り詰め。

## 7. Chat Engine (Ollama LLM)
- **機能**: 最終回答を生成。
- **モデル**: `qwen2.5-coder:7b`
- **接続制約**: 並行実行はせず、シリアル（Max Concurrency: 1）で実行。
- **出力**: ストリーミング出力または一括出力（Typer CLI で表示）。

## 8. Session Manager
- **機能**: マルチターン対話の履歴保持。
- **保存内容**: ユーザー質問と LLM 回答のペア。

## 9. Project Manager
- **機能**: プロジェクトの状態（名前、パス、インデックス状況）を管理。
- **設定ファイル**: プロジェクト一覧を JSON または YAML で管理（例: `~/.java_qa_agent/projects.json`）。
